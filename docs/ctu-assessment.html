Overall Completion Status

CTU has a fully functional engine, two distinct game modes, a Codex, settings, and an onboarding flow. The main gaps are core feel mechanics (damage preview, unit-acted visual, terrain defense), content depth (3 maps is thin), and completely missing audio. It's playable but not yet "enjoyable" by genre standards.

🚨 P0 — Blocking Issues (must fix before testing)

These gaps make the game confusing or feel broken to any first-time player.

1

No "unit acted" visual state

After a unit moves AND attacks, it looks identical to a fresh unit. Players can't tell who has remaining actions. In Advance Wars, spent units are drawn at 50% brightness. This is the most confusing omission — testers will waste turns clicking dead-end units.

Critical UX
UIEngine

Fix: draw unit tokens at 60% alpha when both movedThisTurn && attackedThisTurn.

2

No damage preview before attacking

Players must click an enemy blind with no idea how much damage will be dealt or received. Every tactics game in the genre (AW, ITB, Tacticus) shows a forecast panel or tooltip. Without it, attacking feels like gambling.

Critical UX
UIEngine

Fix: on hover over attackable enemy, show "⚔ -25 HP" preview in HUD selection line or as canvas overlay number.

3

No attack range ring overlay

Blue cells show where a unit can move. But there is no overlay showing the unit's attack reach from current position. Players don't know if they're in firing range until they randomly click an enemy and nothing happens. The red outline only appears on current live targets — it gives no spatial understanding.

Critical UX
Render

Fix: when a unit is selected, draw a secondary translucent orange ring showing all cells within rangeMin–rangeMax Manhattan distance.

4

End-turn button has a dead-code guard (hotseat / CPU)

The end-turn handler contains if (game.currentPlayer !== game.currentPlayer) return; — this condition is always false, making it a no-op. In CPU mode, pressing "End Turn" when the AI is nominally "taking its turn" (during the 400ms setTimeout) can double-trigger the AI. Should guard against double-ending.

Bug
Engine

Fix: track an aiRunning flag, disable End Turn button while AI turn is in progress.

5

No loading state during bootBattle / bootHotseat

JSON fetches take 100–600ms on slow connections. During that time the canvas is blank and the button appears unresponsive. Users will click again, potentially double-booting.

Bug / UX
UI

Fix: disable the triggering button + show "Loading…" in canvas before await Promise.all, re-enable on resolve.

⚠ P1 — Core Fun (needed for enjoyment)

These aren't bugs, but their absence makes CTU feel shallow compared to its genre peers.

6

Terrain defense bonus absent

Terrain moveCost exists and sightCost was added, but defensBonus is not. Every terrain tile in the genre reduces incoming damage: AW gives 10–30% reduction. Forest + hill tiles without defense feel wasted in CTU's current formula.

Core Feel
EngineConfig

Fix: add defenseBonus (0–30%) to tileTextures.json; apply in resolveAttack as damage multiplier.

7

No counter-attack

When you attack a unit at close range, it should fight back at reduced damage (standard in AW, XCOM, etc.). Currently combat is entirely one-sided. This removes the strategic cost of attacking and makes aggressive play overpowered.

Core Feel
Engine

Fix: after resolveAttack, if target is still alive, has direct attackType, and attacker is within target's range, auto-resolve a 50%-damage counter-attack.

8

AI is too weak for any post-Academy challenge

Current AI: move toward nearest enemy, shoot if in range. It never uses terrain, never retreats, never prioritizes weak targets, never flanks around forest LOS. Players who beat the Academy immediately outgrow the AI. The AI is also broken in hotseat-like situations since it only ever tries owner=1 units.

Core Feel
Engine

Fix: add threat scoring (target lowest HP / best damage ratio), add a "retreat if HP below 25%" rule, add indirect-unit positioning logic.

9

Movement is 4-directional only; attack range uses Manhattan

The A* pathfinder and reachable tiles use neighbors4 (N/S/E/W only). Tanks and snipers with range > 1 use Manhattan distance for checking attack range — meaning they cannot attack diagonally at range 2 even though visually the tile is "2 away". This feels like a bug to players expecting standard tactics movement.

Core Feel
Engine

Fix: add neighbors8 for movement + switch attack range check to Chebyshev distance (max of |dx|, |dy|) in combat.js.

10

Zero sound effects

AudioManager.play() is a complete no-op stub. Silence during attacks, movement, and win/loss makes the game feel like a prototype in a demo. Even 4–5 Web Audio API synthetic beeps (no files needed) dramatically improve feel.

Core Feel
Audio

Fix: implement Web Audio API synth sounds for UnitSelected, MoveStarted, AttackImpact, Victory, Defeat — no asset files required.

11

No per-scenario completion tracking

Only academyComplete is tracked. The "Hold the Line" trial has no completion flag; replaying it doesn't unlock anything. Players have no reason to replay. A scenarioWins map in progress + unlock-on-beat would close this.

Unit roster gaps

Missing from the plan's roster A: Sapper (demolish terrain/obstacles), Decoy (absorbs AI targeting), Jet Bomber (AOE indirect). The roster B CSV also listed Rocket Launcher, Anti-Air, and Recon Jeep — none exist in units.json yet.

🔊 Audio — Zero Implementation

Every AudioManager.play() call in main.js is a no-op. The current implementation:

const sounds = {};
export const AudioManager = { play(name) { void sounds[name]; } };

This is the single biggest quality-of-life gap. Even simple synthesized sounds would elevate the game significantly.

Option A — Web Audio API synth (no files, immediate)

Generate all needed sounds procedurally using the browser's built-in Web Audio API. Zero asset downloads, always works offline. Best for MVP testing.

Option B — Free CC0 audio packs

Kenney.nl (kenney.nl/assets/interface-sounds) provides CC0 WAV/OGG packs. Drop files into attached_assets/audio/ and wire paths in AudioManager.

Option C — Mix

Synth for immediate sounds (select, move), Kenney for impact and music. Best result with minimal effort.

Portraits

Placeholder portraits were generated by the Python script (tools/gen-brand-assets.py) and exist at attached_assets/units/*.png. They show initials on colored circles. Real circular cinematic portraits can replace these at any time by overwriting the files with the same names.

VFX PNGs

BattleVfx expects attached_assets/vfx/muzzle_flash.png, missile.png, bazooka_shot_effect.png. Without them VFX silently skip (no crash). Source assets were referenced in the original plan under the Cursor project assets/ folder but were never copied into the game repo.

Hero banner

assets/hero-cover.png exists as a generated gradient. Real promo art can replace it.

🧪 Testing Plan

No automated tests exist. The following structured manual test pass covers all critical paths.

Pre-flight checks (before any test session)

Serve with HTTP server (not file://)
ES modules require HTTP. Run python -m http.server 8765 from repo root, navigate to http://localhost:8765.
Open browser DevTools console (F12)
Watch for 404 errors (missing JSON/assets) and JS exceptions. Any error before wireUi() means the game won't start.
Clear localStorage before first test
localStorage.removeItem('ctu_progress_v1') in console — ensures clean Academy-locked state.
